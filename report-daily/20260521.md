## Bài học hôm nay

**Chủ đề**: OOP
**Chủ đề**: OOP

**Ghi chép tư duy**:
- OOP không bắt buộc như Java, tức là có thể viết hàm thuần túy, chỉ khi cần mô hình hóa thực tể phức tạp thì cần
- Mọi phương thức instance đều phải có self làm tham số đầu tiên, giống this java
- tạo object không có new
- class attribute giống static field của java, thuộc về class
    - có thể tạo instance attribute riêng, không ảnh hưởng đối tượng khác
- __init__ tương tự constructor java, nhưng python chỉ có 1 constructor, không overload -> dùng default param để xử lí nhiều trường hợp
- Lify cycle đi từ new tới init
    - new: mặc định luôn trả về 1 object mới khi khởi tạo -> khởi tạo object
    - init: khởi tạo state -> các trường trong class
-  __del__: để garbage tài nguyên -> không rcm -> dùng exit
- with ... as giống try with resource của java -> auto đóng kết nối
- Tính đóng gói
    - không dùng từ khóa như Java
    - dùng qua các dấu _ để phân biệt phạm vi
        - không có dấu gì, ví dụ: name: str -> public
        - Có dấu _, ví dụ _salary: float -> protected -> Nội bộ và subclass -> lái lái private không được dùng ngoài
        - Có dấu __ , ví dụ __id: str -> private chỉ trong class -> truy cập sẽ sai -> đặc biệt vẫn truy cập được nếu biết tên class, ví dụ quy tắc: _Employee__id
- getter và setter giống giống java hoàn toàn, kiểu dạng attribute syntax chứ không phải method -> dùng decoration property để getter, @proFunction.setter để setter
- kế thừa dùng syntax class Child(Parent)
- Có override
    - Hơi lạ không có default, thằng cha sẽ raise NotImplementedError, thằng con kế thừa phải triển khai cụ thể
- Có super().method()
- Có đa kế thừa Child(Parent, ...)
- MRO - method resolution order, trái sang phải, con trước cha
- Tính đa hình polymorphism, không cần interface hoặc abstract class -> Không cần khai báo interface, chỉ cần cùng tên phương thức
- Duck Typing: nếu nó đi như vịt và kêu như vịt thì nó là vịt -> python không kiểm tra kiểu, chỉ kiểm tra hành vi
- Có abstract method từ module abc dùng @abstractmethod
- __str__ và __repr__
- có abstract class dùng decoration @abstractmethod
- Dunder method là magic method, python gọi tự động trong các tình huống cụ thể, tương tự toString, equals, hashcode... của java
    - Khi một class định nghĩa một số dunder methods đặc biệt, object tạo từ class đó có thể hoạt động giống các kiểu dữ liệu built-in của Python, ví dụ: có thể lặp như iterable, dùng len(), so sánh, truy cập phần tử bằng [], cộng/trừ, in ra đẹp hơn, v.v.
    - Thay vì như java v1.add(v2), v2.subtract(v1), python cho phép dạy object hiểu toán tử + - * /
- Dataclass giống lombok trong java
    - tự sinh init, repr, eq
    - có thể tùy chỉnh
    - Dùng dataclass khi class chủ yếu để lưu dữ liệu
    - Dùng class thường khi class có logic phức tạp, cần kiểm soát chặt init, validation đặc biệt trong get set..., kế thừa phức tạp
    - Frozen dataclass - immutable dùng làm dict key hoặc trong set -> không chỉnh sửa data sau khi init -> có hash
