## Bài học hôm nay

**Chủ đề**: Xử lí file

**Ghi chép tư duy**:
- with as luôn dùng khi open file tương đương try-with-resource bên java -> file tự đóng khi ră khỏi with block dù có exception, ví dụ: with open("file.txt", "r", encoding="utf-8") as f:
- Có thể đọc toàn bộ file thành 1 string với read
- Có thể đọc theo dòng với readline()
- Có thể đọc theo số kí tự n với read(n)
- Có thể đọc toàn bộ nội dung thành 1 mảng với readlines()
- Có thể duyệt file lazy không load vào ram -> object trực tiếp
- không dùng with phải đóng thủ công -> không recommend -> phải dùng try finally để đóng với f.close
- Có nhiều chế độ mở file, tham số thứ 2 trong open
    - r -> đọc, lỗi nếu không tồn tại
    - w -> ghi mới, xóa nội dung cũ nếu tồn tại, tức nó sẽ bị ghi đè -> vẫn tạo mới khi không tồn tại trước đó
    - a -> append nội dung, không ghi đè -> vẫn tạo mới khi không tồn tại trước đó
    - x -> exclusive create -> tạo mới file, lỗi nếu đã tồn tại
    - r+ -> Vừa đọc vừa có thể ghi theo kiểu append -> không overwrite nội dung cũ
    - **Chế độ binary**
        - rb là đọc binary
        - wb là ghi binary
    - Luôn chỉ định encoding khi làm việc với text -> mục đích khi không đúng encoding khi mở sẽ ra kí tự confuse
- Con trỏ file 
    - tell() -> check con trỏ file hiện tại đang ở đâu
    - seek() -> điều khiển con trỏ file
- Mặc định python đã dùng buffer 8192 bytes, tức là ghi tối đa được bao nhiêu kí tự 1 lần
- StringIO -> file ảo trong memory -> có thể dùng build string lớn hiệu quả hơn += tương tự StringBuilder java
- Ghi file binary, ví dụ ảnh
- Đọc parse JSON
    - cần import json
    - cần serialize thành string, dùng dumps(data) -> data ở đây là 1 dict
    - có thể giữ nguyên kí tự unicode với tham số ensure_ascii=False -> tức không bị minimize space
    - có thể parse json dùng loads(json_str) -> thành dict -> thao tác như dict bình thường
- Đọc ghi file json
    - Khi ghi, vẫn dùng dump và truyền qua tham số, json.dump(employees, f, ensure_ascii=False, indent=2)
    - Khi đọc vẫn dùng load, loaded = json.load(f)
- Xử lí kiểu Python mà Json không hỗ trợ, có kiểu {}, kiểu datetime -> cần tạo 1 lớp kế thừa json.JSONEcoder, định nghĩa hàm default
- Xử lí CSV với python
    - ghi csv vào file với writer và writerrows (nhận 1 list)
    - Đọc file csv ra dùng reader và next -> next dùng ghi quét từng row
- Có đọc và ghi với dict
    - Bao gồm DictReader và DictWriter -> đọc ghi theo tên cột
- Module - 'import', java tương đương import com.example.MyClass
    - import toàn bộ module dùng import
    - import cụ thể tử module dùng from ... import
    - Alias -> đổi tên khi import
    - from <module> import * -> tránh dùng trong code production
- Các module có sẵn quan trọng
    - os -> tương tác với hệ điều hành
        - lấy thư mục hiện tại getcwd
        - đổi thư mục chdir
        - lấy danh sách file thư mục listdir(".")
        - tóm lại các command có ở linux, window hầu như có hết tại đây
    - pathlib là cách hiện đạiiad hơn os.path -> dùng OOP thay vì hàm
- datetime xử lí thời gian
- json
    - loads -> loads string thành obj tức dict
    - dumps -> obj thành string
- re -> module xử lí regex
    - tìm kết quả theo regex
    - thay thế
    - tách chuỗi theo regex
- 1 module = 1 file .py
- 1 package = thư mục có file __init__.py
- Try except tương đương với try catch java, không có checked exception như java
    - Có block logic chỉ chạy khi try thành công -> else block
    - raise và re-raise giống throw của java
    - Có bắt nhiều loại exception dùng tuple trong khối except
    - Bắt tất cả exception dùng Exception -> tương tự java
- Raise và re-raise
    - re raise log rồi throw lại exception gốc, không nuốt
- Context manager và Exception
