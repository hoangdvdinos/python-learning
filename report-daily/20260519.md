## Bài học hôm nay

**Chủ đề**: Hàm & Iterator

**Ghi chép tư duy**:
- def trong python không có kiểu trả về, nếu không tường minh mặc định trả về None
- có thể return nhiều giá trị, ngầm định là tuple
- Các cách truyền đối số
    - mặc định positional, truyền theo thứ tự giống java
    - truyền theo tên tham số: describe_product(price=25_000_000, stock=10, name="Laptop")
- xử lí tham số
    - default value: đặt khi định nghĩa def,vd: def create_user(name, role="user", is_active=True)
- positional-only và keyword-only
    - 2 khái niệm về chỉ được phép sử dụng đúng vị trí hoặc keyword params, vd def connect(host, port, *, timeout=30, retry=3) -> tham số sau *, bắt buộc phải gọi bằng tên
    - def normalize(value, /, min_val=0, max_val=1) -> tham số trước /, bắt buộc nhận theo postional(không phải keyword param)
- *args và **kwargs
    - *args tương tự String spread operator của java
    - kiểu mặc định sau khi nhận vào là tuple (iterable)
    - có thể unpacking với * sau khi gọi hàm
    ```
    def add(a, b, c):
        return a + b + c
    values = [1, 2, 3]
    print(add(*values))
    ```
    - **kwargs 
        - java tương đương Map<String, Object> 
        - Tức là truyền theo danh sách keyword-only
        - Hoặc truyền vào 1 dict với dạng unpacking \*\*, vd: create_user(**data)
- Thứ tự tham số chuẩn
```
# Thứ tự bắt buộc: positional → *args → keyword-only → **kwargs
    def full_example(pos1, pos2, *args, kw_only=0, **kwargs):
        print(f"pos: {pos1}, {pos2}")
        print(f"args: {args}")
        print(f"kw_only: {kw_only}")
        print(f"kwargs: {kwargs}")

    full_example(1, 2, 3, 4, 5, kw_only=99, extra="hello", flag=True)
    # pos: 1, 2
    # args: (3, 4, 5)
    # kw_only: 99
    # kwargs: {'extra': 'hello', 'flag': True}
```
- Có lambda expression giống Java
    - lambda chỉ được phép 1 expression, không có statement, không có return
    -> tức là trong body chỉ có 1 expression
    - labda thường dùng làm argument cho các hàm bậc cao
