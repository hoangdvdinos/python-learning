## Bài học hôm nay

**Chủ đề**: Hàm & Iterator

**Ghi chép tư duy**:
- def trong python không có kiểu trả về, nếu không tường minh mặc định trả về None
- có thể return nhiều giá trị, ngầm định là tuple
- Các cách truyền đối số
    - mặc định positional, truyền theo thứ tự giống java
    - truyền theo tên tham số: describe_product(price=25_000_000, stock=10, name="Laptop")
- xử lí tham số
    - default value: đặt khi định nghĩa def,vd: def create_user(name, role="user", is_active=True)
- positional-only và keyword-only
    - 2 khái niệm về chỉ được phép sử dụng đúng vị trí hoặc keyword params, vd def connect(host, port, *, timeout=30, retry=3) -> tham số sau *, bắt buộc phải gọi bằng tên
    - def normalize(value, /, min_val=0, max_val=1) -> tham số trước /, bắt buộc nhận theo postional(không phải keyword param)
- *args và **kwargs
    - *args tương tự String spread operator của java
    - kiểu mặc định sau khi nhận vào là tuple (iterable)
    - có thể unpacking với * sau khi gọi hàm
    ```
    def add(a, b, c):
        return a + b + c
    values = [1, 2, 3]
    print(add(*values))
    ```
    - **kwargs 
        - java tương đương Map<String, Object> 
        - Tức là truyền theo danh sách keyword-only
        - Hoặc truyền vào 1 dict với dạng unpacking \*\*, vd: create_user(**data)
- Thứ tự tham số chuẩn
```
# Thứ tự bắt buộc: positional → *args → keyword-only → **kwargs
    def full_example(pos1, pos2, *args, kw_only=0, **kwargs):
        print(f"pos: {pos1}, {pos2}")
        print(f"args: {args}")
        print(f"kw_only: {kw_only}")
        print(f"kwargs: {kwargs}")

    full_example(1, 2, 3, 4, 5, kw_only=99, extra="hello", flag=True)
    # pos: 1, 2
    # args: (3, 4, 5)
    # kw_only: 99
    # kwargs: {'extra': 'hello', 'flag': True}
```
- Có lambda expression giống Java
    - lambda chỉ được phép 1 expression, không có statement, không có return
    -> tức là trong body chỉ có 1 expression
    - labda thường dùng làm argument cho các hàm bậc cao
    - bản chất dùng logic 1 def vào 1 def chính -> linh hoạt, tái sử dụng
- enumerate trả về 1 iterable có index
- zip ghép song song giá trị các mảng, mặc định dừng khi mảng ngắn nhất kết thúc
- scope biến
## Question bản thân**
- Sao python nó lại linh động về tham số trong function thế nhì ?
-> T nghĩ nó linh động này thì code sẽ rất là đa dạng về ý tưởng, vì python và ngôn ngữ bậc cao, rất nhiều built-in function, và code ngắn, linh hoạt, bỏ nhiều cái.
- Tại sao sinh ra lambda ?
-> lambda giải quyết việc thay vì định nghĩa 1 def hoàn chỉnh dài, chúng ta chỉ cần định nghĩa logic ngắn và nó sẽ được gọi trong fuction chính
- Scope biến: local, global, nonlocal
    - Có thể sửa biến global từ trong hàm
    ```
    def increment():
    global counter
    counter += 1
    ```
    - Có nonlocal sửa biến của hàm ngoài bao quanh gần nhất
    ```
    def make_counter():
    count = 0

    def increment():
        nonlocal count  # Sửa biến count của make_counter
        count += 1
        return count

    return increment    # Trả về hàm — closure!
    ```
- Iterator và giao thức __iter__, __next__
    - tự build cần init, iter, next
- Generator function và yield, mục đích, hold state không đẩy trực tiếp vào ram luôn, mỗi lần gọi sẽ nhảy vào yield 1 lần
- Generator expression dùng để tạo dữ liệu từng phần tử một qua vòng lặp, thay vì tạo sẵn toàn bộ list trong bộ nhớ, syntax dùng (...) -> tiết kiệm bộ nhớ
- Decorator là hàm bọc bên ngoài hàm khác
